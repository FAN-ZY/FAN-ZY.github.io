  view

一、什么是软件实施

解释

通过技术、功能、操作等方面进行系统安装调试、人员培训、上线使用、后期维护等一系列的工作。

地位

软件开发：注重软件的功能和质量

软件实施：使用体验和效果

岗位职责

1、负责公司软件产品系统的安装、配置、使用、维护以及性能的分析、优化、故障处理等相关工作；
2、协助项目经理进行项目实施推进与现场沟通协作，以及控制项目需求，进行线上、线下客户使用培训及答疑；
3、负责驻场实施软件安装与调试；协助项目经理完成调研、上线问题处理等；
4、能够独立处理项目相关的数据实施工作；
5、协助项目经理进行实施进度管理，完成实施相关的文档整理，配合项目验收相关工作；
6、负责项目实施相关文档的撰写，整理，收集客户需求，并给与产品反馈，优化产品升级。
7、监控并汇报项目现场情况，收集用户反馈意见，巩固客户关系，提高客户满意度。

岗位区别

实施工程师 实施顾问
关注点 需要熟悉项目的技术细节，理解系统的整体架构，在项目实施过程中，更多从事系统架构安装、软件调试、程序编写等工作 需要关注客户的需求，在项目实施过程中，主要负责项目整体规划和方案设计，针对不同行业、不同企业的实际业务，制定相应的解决方案
工作流程 承担技术的实际操作，比如安装和调试系统、程序编写和代码调节等任务 承担实施的管理和规划，负责项目的前期分析和调查，方案和需求的制订以及方案的实施计划等任务
岗位要求 技术培训、项目部署和软件开发等方面的能力
对技术要求更高		 方案设计、项目管理和客户沟通等方面的能力

二、实施过程中的具体工作

①投标阶段：

寻找投标机会、购买/制作标书、标书打印/投递、材料存档

向用户移交项目，包括：软件产品、项目实施过程文档、软件项目知识成果的转移等；《用户满意度调查》、项目实施各阶段的评价等；依据合同协同销售及项目经理追收项目尾款。

①网络：

协议

UDP（User Datagram Protocol，用户数据报协议），是一个简单的传输层协议。UDP不保证数据包会到达其最终目的，不保证各个数据包的先后顺序，不保证每个数据包只到达一次。
特性
1.无连接
UDP提供无连接服务，UDP服务器和客户端之间不必存在任何长期关系。
2.传输效率高
UDP 没有可靠性保证，但因此 UDP 在数据传输中有着延迟小、传输效率高的特性。
3.面向报文
UDP 是面向报文的，与 TCP 面向流相对，UDP 会保留报文的边界信息。

TCP UDP
面向连接 面向数据报
可靠 不可靠
较慢 传输速度快

命令

  • ping 命令是用来探测本机与网络中另一主机之间是否可达的命令，如果两台主机之间ping不通，则表明这两台主机不能建立起连接。ping是定位网络通不通的一个重要手段。
  • telnet 是一种Internet进行远程登录的必备协议。telnet协议是属于tcp中的一种协议。使用这种协议，就可以实现在本地计算机上远程登录控制服务器的能力。
  • route 命令可以在数据包没有有效传递的情况下，利用route命令查看路由表；如果traceroute命令揭示出一条异常或低效白传输路径，则可以用route命令来确认为何选择该路径，而且可以配置一个更有效的路由。
  • Netstat 是控制台命令，是一个监控TCP/IP网络的非常有用的工具，它可以显示路由表，实际的网络连接以及每一个网络接口设备的状态信息。
  • Isnrctl 命令是作为oracle监听命令。

文件权限

chmod（change mode）命令是控制用户对文件的权限的命令。

Linux/Unix的文件调用权限分为三级：文件所有者（Owner）、用户组（Group）、其它用户（Other Users）。

chmod可以使用八进制数来指定权限，无需再使用指定的权限和用户的字母来进行标识，通过读写执行等3个权限的数字来进行设置。

chown 命令，可以认为是”change owner”的缩写，主要用于修改文件（或目录）的所有者。除此之外，这个命令也可以修改文件（或目录）的所属组。

3、常见问题

1.在做SQL备份时应该先开启什么服务？
要开启SQLServer Agent，这种服务主要是用于在设定的时间备份、复制数据，以及在自动执行调度表上设置的其他项目。启动这个服务后，设定好在什么时候做什么事，这个服务会让它自动运行，不需要人工干预。

2.系统启动后，不能连接数据库，可能是哪方面的原因？
和数据库有关的服务没启动
防火墙可能阻挡了数据库的端口
数据库可以启动，而登录不了，可能是密码错误和连接参数配置错误

3.一般数据库日志满了，会出现什么情况？是否还能使用？
数据库日志满了，就不能使用数据库，数据库日志满是指数据文件达到设置的最大文件大小，没设置的时候最大可用磁盘空间，只能执行查询，读取操作，不能执行更改和备份操作，原因是写任何操作都要记录日志，也就是处于不可用状态

join 正关联 select * from 表1 inner join 表2 on 表1字段=表2字段； inner 可不写
left join 左关联 select * from 表1 left join 表2 on 表1字段=表2字段；
左侧表为主表，信息全列出，右侧表列出匹配信息，不匹配的返回null记录补充
right join 右关联 select * from 表1 right join 表2 on 表1字段=表2字段；
右侧表为主表，信息全列出，左侧表列出匹配信息，不匹配的返回null记录补充
full join 全关联 select * from 表1 full join 表2 on 表1字段=表2字段；
l两侧表信息全部列出以匹配格式返回，不匹配的返回null记录补充
数据类型 释义 特点
Varchar2 字符类型 4000字节，英文字符4000个，汉字2000个
Number 数值类型 P：Precison，即精度缩写，表示有效数字的位数，最多不能超过38个有效数字
S：Scale，Scale为正数时，表示从小数点到最低有效数字的位数。
Date 日期类型 存储日期和时间信息
Clob 存储单字节和多字节字符数据。支持固定宽度和可变宽度的字符集。
如需存储字符超过4000个，就需要用CLOB类型进行存储。
Blob 存储非结构化的二进制数据大对象，它可以被认为是没有字符集语义的比特流;
一般是存储图像、声音、视频等文件。

SQL约束

约束用于限制插入表的数据。
可以在创建表时规定约束《通过CREATE TABLE 语句），或者在表创建之后也可以（通过ALTER TABLE 语句进行修改）。我们将主要探讨以下几种约束：

约束 释义 范例
NOT NULL 强制列不能为空 create table 表名 (字段名 number(10) not null);
UNIQUE 约束唯一标识 alter table 表名 add unique (字段名);
注：每个表可以有多个Unique约束，但是每个表只能有一个Primary Key 约束。
PRIMARY KEY 主键，唯一非空 create table 表名 (字段名 number(10) PRIMARY KEY);
alter table 表名 drop constraint 主键名； 删除主键
每个表应有且只有一个主键
FOREIGN KEY 外键 create table 表名 (字段名 number(10) references 外键字段名);
alter table 表名 drop constraint 主键名； 删除主键
alter table 表名 add FOREIGN KEY (字段名) references 外键字段名;
CHECK 限制列值范围 create table 表名 (字段名 date check
(字段名>to_date(‘2024-06-18’,’yyyy-mm-dd’) ) );
如果对单个列定义 CHECK 约束，那么该列只允许特定的值。
DEFAULT 插入默认值 create table 表名 (字段名 date DEFAULT sysdate); 插入当前时间
alter table 表名 modify 字段名 default null； 取消默认值
如果没有规定其他的值，那么会将默认值添加到所有的新记录。
函数 释义 语法
null 列值不设定，默认为null 条件查询时，用“字段 is null”查询；用“字段=’ ‘ ”是查询不到的
NVL() 从两个表达式返回一个非 null 值 NVL（eExpression1，eExpression2）
1为null返回2；1不为null返回1；1、2都为null则返回null.
Upper() 字段值转换为大写 select upper(字段) from 表名；
select * from 表名 where upper(字段名)=upper(‘条件值’)；
insert into 表名 values（值1，upper（’值2‘），值3，……）；
Lower() 字段值转换为小写 select lower(字段) from 表名；
用法与upper()一样
Substr() 截取字段位数 select substr(字段,起始位数，截取个数) from 表名；
Length() 字段长度 select * from 表名 where length(字段)=10；
Round() 精度控制 select round(字段,精度位数) from 表名；
To_char() 字符格式转换 select to_char(时间字段) from 表名 ；
select to_char(时间字段,’yyyy-mm-dd’) from 表名 ；
聚合函数 释义 语法
Avg 取平均值 select avg(字段) from 表名；
Count 统计总数 select count(*) from 表名； null值不计数
select count(distinct 字段) from 表名； 去重后统计
Max 取最大值 select max(字段) from 表名；
Min 取最小值 select min(字段) from 表名；
Sum 计算总和 select sum(字段) from 表名；

四、项目实施工作标准化流程

项目启动阶段

主要任务：

l公司和客户双方选定合适的项目经理，分别成立项目实施小组和客户实施小组；

l与客户实施共同商讨制定项目《项目实施主体计划书》，规划出整个项目的实时进程；

l与客户实施组共同研讨制定实施项目章程，明确项目实施过程中涉及组织、范围和权责的相关约定；

l公司和客户双方项目经理共同推动并组织召开项目启动会。使客户的高层及项目组全体成员对项目实施的方式和步骤有清楚的认识，建立相互合作、共同推进项目实施的基础。

1、项目交接

****工作任务****：（内部交接——销售向实施交接）

1）客户经理分类准备售前阶段所有与该项目有关的信息和资料；

2）销售部门准备申请实施的资料，向实施团队发起销售转实施交底；

3）客户经理组织项目相关人员举行项目内部交接会议；

4）记录并整理会议纪要与会双方负责人签署会议备忘录。

****工作流程****：

*①**项目实施的申请*

准备《项目售前调研报告》、《投标技术方2案》、《实施服务合同》等复本

准备客户方所有接触过的重要人员的直接信息和分析资料

整理销售人员、售前顾问对客户做的口头承诺的内容清单

严格遵照制度编写《销售转实施交底单》，提出正式的项目实施申请

*②举行**内部交接会议*

准备会议议程，让参会人员明确会议主题、会议内容、参会人员、会议时间等相关信人息

会议上要介绍客户方的人员、企业、产品等方面的信息，让项目实施小组尽可能多的了解项目的信息

*③**内部交接会议答疑*

客户经理或售前顾问有责任将合同中重要的条款部分在此说明（如：实施范围、实施周期、实施内容、收款条件等）

客户经理或售前顾问必须向项目小组介绍在售前阶段向客户口头承诺的内容（如：客户期望、客户化的问题，免费给客户做的事情等）

*④**项目风险的重新评估*

回顾该项目的风险评估报告，客户经理或售前顾问有责任把项目可能遇到的风险在会议上提出。

项目小组根据已掌握的信息及会上所涉及到的内容重新评估风险（如：口头承诺、对合同内容分析出的周期风险等）

重新评估风险分析报告中的风险等级、没有提出的隐形风险等内容，应对策略，修订风险分析报告

*⑤**会议备忘录的签订*

项目小组成员必须记录交接会议纪要，特别是口头承诺、疑问解答内容，参与人共同确认

项目经理和客户经理在会议后必须签署《项目内部交接会议备忘录》

2、项目筹备

*工作任务：*

1）项目经理拜会客户项目负责人；

2）项目经理同客户高层进行深入沟通；

3）项目经理帮助企业组建实施小组。

​ *工作流程：*

*①**拜会客户项目负责人*

项目经理拜会企业的项目前期负责人，获取企业对项目的重视程度和企业内部支持与反对等信息

项目前期负责人确定拜会客户高层的相关事宜

*②**拜会企业高层*

与企业高层的首次沟通的目的是要与企业高层讨论项目实施的目标、范围、验收标准，摸清企业高层对项目的重视程度、预计投入的相关人员等情况

明确企业在项目整个实施中主导的作用，企业高层应该如何推动项目的实施，需要做哪些具体工作

与企业高层明确项目的实施方式，要求客户方必须组建项目实施小组，与公司实施小组共同完成整个实施工作，提出对项目小组的相关要求

*③**成立客户实施小组*

协助企业高层选定合适的项目经理

向企业高层解释项目组成员在项目实施中的重要作用，提议哪些部门的人员参与到项目实施小组中

要求企业配备系统管理员、技术顾问人员，明确他们在未来的项目实施中的关键作用，帮助企业选定项目组其他成员

*④**建立项目领导小组*

项目经理根据项目情况，若客户项目实施小组成员中没有企业高层直接参与，必须建议客户组建项目领导小组，高层人员必须加入其中，负责解决项目实施过程中出现的有争议的问题

建立项目领导小组的同时必须明确问题汇报与研究解决的机制，保障领导小组能及时研究、决策、协调、落实问题

3、实施计划指定

*工作任务：*

1）项目经理与客户项目经理商定实施目标；

2）项目组进行补充调查，进一步了解客户的实际情况；

3）编制《项目实施章程》，规划整个项目的实施过程；

4）客户项目经理负责将《项目实施章程》提交企业高层进行审批。

*工作流程：*

*①**明确项目实施目标*

按照《产品销售合同》、《实施服务合同》中规定的项目实施目标和范围与客户项目经理进行沟通，主要包括：组织范围、功能范围、业务流程范围、二次开发范围、技术实施范围、培训范围等

如果对上述内容还存在有争议的内容，经双方项目组协商一致后，做补充说明

若沟通中发生客户项目经理无法认定的内容，应共同与领导小组（企业高层）沟通确定，并以书面形式记录

*②**企业的补充调查*

项目经理必须对于企业关键业务环节和客户非常关注的地方向客户项目经理及客户项目小组进行补充调查，这种调查可以采用现场访谈、企业介绍等方式。

进行调查时，切忌与企业讨论过深的问题，主要是了解为主，清楚地知道企业内部人员对项目的支持情况、企业业务流程中存在的弊端，企业急于想解决的问题、项目支持者对项目实施所担心的问题

企业在这个阶段一般是不会有什么保留的，他们会尽量的满足我们的需要并暴露出企业上项目的真实目的

*③**规划项目实施的过程*

根据项目的大小和实施周期的长短，以及项目实施的风险和复杂程度做出分析每个阶段中必须明确工作任务、完成标志，客户方和公司方的职责分别是什么，项目验收的基本标准、验收策略

每个阶段中必须明确工作任务、完成标志，客户方和公司方的职责分别是什么，项目验收的基本标准、验收策略

*④**协助客户建立项目进度汇报机制*

制定项目例会制度，在项目最初每周至少要开一次项目例会，总结本周的项目实施情况，布置下周的工作安排

建议客户项目负责人必须每月参加一次项目例会，听取项目实施双方的工作汇报，如果遇到特殊问题，需要与项目负责人进行直接的沟通

领导小组要了解每月项目的进展状况，对于影响项目进度的人员要给予相应的处罚，客户项目经理可以给予处罚的建议

明确项目实施过程中有关项目管理方面的具体要求，对于可能出现项目变更要有严格的审批程序，重大变革必须由项目负责人亲自审批。减少项目变更给项目实施带来的不利因素

4、项目启动会

*工作任务：*

1）项目启动会是一次非常好的传达这件事情的机会；

2）让客户方老总向中、高层的管理人员传递企业信息化实施的决心，为我们日后开展相关的工作扫除障碍，让客户方从上到下达成一种共识；

3）双方项目组成员第一次在企业全体亮相。

*工**作流程：*

*①**项目启动会准备*

1）与客户高层商定会议日程，以及需参加人员名单；

2）选定会议主持人、明确会议的议程；

3）准备相关演讲PPT资料；

4）准备领导讲话演讲稿；

5）客户方准备会议场地；

6）会场布置；

7）发出会议通知。

*②**召开启动会*

项目启动会只是一个形式，标志着项目的正式启动，真正重要的是启动会前的准备工作是否充分，启动会的效果是否达到预期的目标，这才是我们双方最关注的地方。因此，项目启动会前必须做充分的会前准备工作，与客户高层深入交流，明确启动会的意义所在。

*③**会议纪要的签字确认*

目的主要是让与会人员再一次在感性上增强认识，引起大家的充分重视，让客户公司的相关人员感受到实施的力度。

5、总结

以上为项目准备阶段要完成的内容，其中公司方面主要派出项目经理、销售顾问。客户方面主要派出项目经理和客户中高层。通过双方相关工作人员的配合，为后续实施开展做好了充足的准备工作，接下来进入第二个阶段，需求调研阶段，我们实施工程师也要登场了。
